Atelier 1 Policy on Generative A.I. Tools v 1.0

Overview

In this course we will actively be exploring how current AI tools can intersect with your creative practice. You don't have to use them, but you should at least understand them. We will primarily be engaging with these tools for coding, but you can also use others.

This policy is broadly based around a simple concept: Completion Is Irrelevant

In most of this course it is possible to almost entirely offload the process of creating the work to any contemporary large language model (LLM) to create a basic, boring outcome that fulfills the assignment brief. If you want to waste your time and $$$ working that way, be my guest. These tools will likely be part of your educational and professional practice from here on, so it is important that you start to figure out how to work with them, not for them.

Guidelines

  • Don't offload your creative practice. These tools are powerful, but tend to dull work into a generic middle.
  • Don't trust them. Design guardrails that keep things from going very wrong
  • Don't think that they are magic. These tools require learning like any piece of software.
  • Be open about how you've worked with them when discussing your work with your peers and professor. We are all trying to figure it out.
  • Treat them as things to explore
  • Find ways to integrate them into your process, not the other way around

Methods

We will be focusing on how you can develop various media to clearly explain and refine your ideas outside of the standard chat format. The goal is to provide you with tools to develop your general skills of Algorithmic Thinking which are critical whether using AI tools or not.

Tools

VS Code + Github Copilot Pro

There are many different coding tools available, but we will be focussing on using VS Code + Github copilot for a few reasons.

  • It is an industry standard toolset
  • You get Free access through OCADU

We will have multiple sessions looking at various aspects of this tool including

  • How to properly set up your production environment
  • How to set up guardrails and backups
  • How to get good at it
  • Looking at multiple types of Interactions
