Canvas Design Agent

Transform plain text into visually rich Canvas LMS pages. No HTML knowledge required.

What This Is

Canvas LMS strips almost all CSS and JavaScript from pages. Only inline style="" attributes and a specific set of HTML elements survive the Rich Content Editor (RCE). This makes it difficult for faculty to create visually structured course pages.

This project provides two things:

  1. SKILL.md -- An instruction file that tells any AI agent (ChatGPT, Claude, Copilot, Gemini, etc.) exactly how to generate Canvas-compatible HTML from your plain text content.
  2. Element Catalog -- A numbered reference of every HTML element and pattern that works in Canvas, with visual previews and code.

Faculty write content as plain text or markdown, reference element numbers if they want specific layouts, and the agent handles all the HTML.

How It Works

You Write

Plain text or markdown describing your course content: class schedules, assignment details, resource lists, announcements, outlines, syllabi.

Optionally reference element numbers: "use L03 for the layout" or "make the deadlines a V03 warning box."

Agent Generates

Complete Canvas-compatible HTML using only inline styles and tested elements. Ready to paste directly into the Canvas HTML editor.

The agent follows the constraints automatically -- you never need to worry about what Canvas strips.

Workflows

The SKILL.md file works with any AI agent. Choose the workflow that matches your setup.

Workflow A: Local Agent (VS Code, Cursor, etc.)

Best for faculty who already use a code editor or want to manage multiple pages in a project.

  1. Set up the skill. Copy SKILL.md into your project's .github/ directory. Most editors with AI assistants (VS Code with Copilot, Cursor, Windsurf) will automatically read instruction files from this location. Consult your editor's documentation for the exact path if needed.
  2. Create a content file. Write your page content in a .txt or .md file. Use plain language -- describe what the page should contain, its structure, and any specific elements you want.
  3. Ask the agent to transform it. In your editor's AI chat, ask it to convert your content file into Canvas HTML. Example prompt: "Transform this into a Canvas page using the skill. Use a gradient header and collapsible sections."
  4. Copy to Canvas. Open your Canvas page, switch to the HTML editor view (click the </> icon in the RCE toolbar), paste the generated HTML, and save.

Tip: Keep your content files organized by course. Create a folder per course and place a course-specific template alongside the content files (see the Methods section below).

Workflow B: Web-Based Agent (ChatGPT, Claude, Gemini, etc.)

Best for quick, one-off page generation without installing anything.

  1. Upload the skill. Start a new conversation in your preferred AI chat. Upload or paste the contents of SKILL.md. This gives the agent all the Canvas constraints and element templates it needs.
  2. Provide your content. In the same conversation, type or paste your page content. Describe what kind of page it is and what elements you want. Example: "Here is my course schedule. Turn it into a Canvas page with a blue header and a schedule table."
  3. Get the HTML. The agent will generate Canvas-ready HTML. Copy the entire output.
  4. Paste into Canvas. Open your Canvas page, switch to the HTML editor view, paste, and save.

Tip: If you have a course-specific template, upload it alongside the SKILL.md so the agent applies your course's colors and layout consistently.

Element Numbering System

Every Canvas-safe element has an ID number. Reference these in your instructions to the agent to specify exactly what you want.

Prefix Category Examples
L Layout Two-column table, flexbox row, CSS grid, centered container
C Content Organization Collapsible sections, info cards, definition lists, checklists
T Typography Styled headings, highlights, blockquotes, keyboard shortcuts
D Data Display Data tables, schedule grids, comparison tables, progress bars
V Visual Indicators Accent boxes, color headers, alert callouts, badges, gradients
N Navigation Anchor links, button-styled links
X Canvas Integration Assignment links, file embeds, video embeds, audio players

Example instruction to agent: "Create a page with a V05 gradient header, the schedule as a D05 grid, and put the resources in a C01 collapsible at the bottom."

Canvas Constraints (Quick Reference)

The SKILL.md contains the full constraint list. Here is a summary of what Canvas does and does not allow.

Works in Canvas

  • All inline style="" attributes
  • Flexbox (display: flex) and CSS Grid (display: grid)
  • Linear gradients
  • Relative and absolute positioning
  • <details>/<summary> collapsible sections (always start closed)
  • <mark>, <abbr>, <ins>, <del>, <kbd>
  • All table features: colspan, rowspan, colgroup, caption, thead/tbody/tfoot
  • Audio and video players (<audio controls>, <video controls>)
  • Definition lists (<dl>)

Stripped by Canvas

These are silently removed or broken when you save a page in Canvas. The agent avoids all of them automatically.

  • <style> blocks, <script> tags, external CSS/JS
  • All SVG elements
  • <meter>, <progress>, <fieldset>, <legend>
  • box-shadow, text-shadow, opacity, transform
  • letter-spacing, word-spacing
  • data: URIs in image sources
  • The open attribute on <details>
  • The reversed attribute on <ol>

Methods: Course-Specific Templates

The element catalog provides building blocks. A course-specific template combines those blocks into a consistent design that you reuse across an entire course.

Why Create a Template?

Without a template, every page is a one-off. With a template, you define your course's visual identity once and every page automatically follows it. This means:

  • Consistent look across all class pages, assignments, and modules
  • Less work per page (the agent already knows your preferences)
  • Easy to update: change the template and regenerate pages

What Goes in a Template?

A course template is a plain text file that describes your preferences. It tells the agent which elements to use and how to style them. Include any or all of these:

Setting What to specify Example
Colors Primary, accent, background, text colors Primary: #2c3e50, Accent: #e74c3c
Header style Which V-series element, colors, content V05 gradient header, primary-to-dark
Page sections Standard sections that repeat on every page Agenda, Content, Resources, Next Class
Layout Column layout, max-width, general structure L05 centered at 800px
Section style How to present recurring sections Agenda as C06 ordered list, Resources as C01 collapsible
Page types Different templates for different page kinds Lecture pages = blue header, Labs = orange header

Save this as a text file (e.g., course-template.txt) alongside your content:

Header with class number, title, and date 2. "Today's Agenda" -- C06 ordered list 3. Main content area (varies per class) 4. "Resources" -- C01 collapsible, contains link list 5. "Next Class Preview" -- C01 collapsible Layout: L05 centered container, 800px max-width Callouts: V01 left-border accent box using accent color Important dates: T02 highlighted in yellow

How to Use a Template

With a Local Agent (VS Code, etc.)

  1. Save your template file in the project alongside SKILL.md.
  2. When asking the agent to generate a page, reference both: "Using the course template in course-template.txt, transform today's class notes into a Canvas page."
  3. The agent reads the skill for Canvas constraints and your template for course-specific design decisions.

With a Web Agent (ChatGPT, etc.)

  1. Upload both SKILL.md and your template file at the start of the conversation.
  2. Provide your page content and ask for a Canvas page.
  3. The agent applies both the Canvas constraints and your template preferences.

Building a Template Library

Over time, you can build a library of templates for different purposes:

  • Course template -- Your main per-course design (colors, headers, standard sections)
  • Assignment template -- Standardized layout for assignment pages (objectives, requirements, rubric, submission instructions)
  • Syllabus template -- Structure for course outlines with collapsible policy sections
  • Schedule template -- How to present the course calendar (D05 grid with color-coded rows by type)
  • Project brief template -- Description, milestones, deliverables, grading breakdown

Each template is a plain text file. The more specific your template, the more consistent the output.

Iterating on Templates

  1. Start simple. Define colors and a header style. Generate a few pages.
  2. Review in Canvas. Paste the generated HTML and see how it looks.
  3. Refine. Add or change template preferences based on what you like.
  4. Expand. Add page-type variations (lecture vs. lab) and section templates.
  5. Reuse. Next semester, update dates and topics -- the template stays the same.

Getting Started

Ready to build your first Canvas page? Follow these steps:

  1. Browse the Element Catalog. Look through the available elements and note any you want to use on your page.
  2. Write your content. Open a text editor and describe your page in plain language. Include the actual content (text, dates, links) and optionally note which elements to use.
  3. Choose a workflow. Decide whether to use a local agent or web-based agent.
  4. Generate and paste. Use the agent with SKILL.md to transform your content, then paste the HTML into Canvas.

First time? Try a simple page first: write a paragraph with a heading and ask the agent to format it with a V02 color header and a C01 collapsible section for additional notes. Once you see how it works, build more complex pages.
