Introduction

Create a custom controller for the classic arcade game Pong, which was first released in 1972. The controls for the game are quite simple: you can move a 2d paddle up or down to reflect the bouncing ball. You will develop a new and novel controller for the game that examines and pushes what is possible with this seemingly simple gameplay. Do not copy existing controller types or look to controllers that simply make the game easier (that is boring...), instead you should create an object, objects, or installation that playfully explore new possibilities for this classic game. 

Digital Inputs

One of the most common first steps when working with Arduino, is to use learn the digitalRead() command. This command allows your program to read whether or not a specified pin is receiving voltage or not and converts the information to either:  0 (No Voltage) or 1 (Voltage). This action is typically associated with reading whether or not a button is being pressed, but for this project you are to push beyond a standard button to understand what else can be sensed when you create your own digital sensor. Digital sensors can be created from any material or object that can conduct electricity.

A few examples of what these types of sensors can detect

  • Presence/Absence of a person or object
  • Speed / Rates of Change
  • Direction

DF Pong

Your controller will connect to a custom version of the game that was created for this project : DF Pong. This version of the game connects with your Arduino Uno R4 Wifi via Bluetooth to allow any 2 controllers to compete in a match. The game uses a very simple protocol to control the movement

  • 1 - UP
  • 0 - DON'T MOVE
  • 2 - DOWN

This connection is enabled by the DFPongController library that handles the connection with the game and sending the control data

Design Constraints

Some basic and rigid constraints that will improve your controller

  • Do Not copy existing Game controller formats.
    • This includes both the form of the controller and inputs
    • Don't just re-create existing buttons
  • Do Not use any off the shelf buttons - You must make your own inputs
  • Wire your own LED Indicator to work with the library's setStatusLED() method
  • Do Not base the controller on stuffed animals or weapons (both are boring for their own reasons)
  • Create the controller with a robust design that will allow it to be used with the Pong Game

Design Goals and Considerations

Push the limits of what can be even considered a controller, but also

  • Consider the overall craft and finish of the design
  • Consider how the material, mass, and scale of the controller can impact it's use and gameplay
  • Avoid simply sticking it inside an existing box. Fabricate it yourself
  • Look to fully encase the Arduino, wiring, battery, and sensors
    • Don't glue it together. Use connection methods that allow access
  • Avoid loose and tangled wires
  • Create electrical and material connections that will hold up to use
  • Integrate an indicator LED that displays the controller's connection state
