Introduction

Create an interactive object or wearable that establishes a clear interaction for 1-3 people based around a preposition. Use either the Distance Sensor, a custom pressure sensor, or both as input. The Arduino's 12x8 LED matrix provides visual feedback.

During the exhibition, each work will display its chosen preposition. Visitors will experience your piece and judge whether the interaction clearly embodies that word.

Interaction as Preposition

You are creating a relationship between the visual information on the LED Matrix and a person or thing. Describe that relationship through a simple preposition.

Recommended Prepositions:

Distance Sensing toward/away, between, above/below, through
Pressure Sensing against, within, upon
Both beside, around

Examples:

An object displaying agitated pixels when a hand hovers ABOVE it, calming when they step away. A pouch showing pixels crawling TOWARD the center when squeezed. A tabletop object tracking movement THROUGH its sensing zone.

Hardware

Inputs (use one or both):

  • Distance Sensor (HC-SR04) - Ultrasonic sensor, 2cm to 400cm range. Detects presence, proximity, and movement without contact.
  • Custom Pressure Sensor - DIY sensor using conductive fabric, Velostat, or foam. Requires physical contact. Material choices affect engagement.

Output:

  • LED Matrix - 12x8 red LEDs (on/off only). Control via TinyFilmFestival Library. Create animations in LED Matrix Editor.
Design Requirements
  • Interaction must be defined by a preposition you choose
  • All electronics enclosed (max 30cm x 30cm footprint). Free-standing, tabletop, or wearable
  • LED Matrix covered with magnifier, diffuser, or both. No bare LEDs visible
  • Animation must be "alive" at all times - never static, even when idle
  • Design at least 2 animation states
  • Write your own code (examples as starting point only)

Do Not:

  • Create static displays or simple on/off indicators
  • Make graphs or data visualizations of sensor values
  • Replicate phone/tablet interfaces
  • Embed screen in a stuffed animal
  • Copy a Tamagotchi directly (inspiration is fine, replication is not)
  • Leave wiring exposed
  • Create interactions requiring verbal explanation
Design Considerations

Aliveness:

Design idle animations (breathing, pulsing), acknowledgment when sensing begins, and graceful recovery after interaction ends. The object should always communicate it is ready and responsive.

Dialogue:

What is the sequence of action and response? How does the object invite interaction, acknowledge input, respond, and invite the next action? Design this conversation intentionally.

Affordances:

How does someone understand how to interact without instructions? Consider material, scale, sensor orientation, and how to make the sensing zone discoverable.

Craft:

The screen is very small - consider intimate scale. Link animation to material choices. Build robust connections for exhibition handling. Make material choices that reinforce your preposition.
