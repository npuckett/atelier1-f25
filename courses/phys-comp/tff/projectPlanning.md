# Notes for planning your Tiny Screens Project
This project takes a sizeable technical jump from project 1 since you must deal with *both* the input and the output as a cohesive interaction concept. Below are some general notes but please carefully read through the project description : https://canvascloud.ocadu.ca/courses/13538/pages/tiny-screens

## Actually start by picking one of the prepositions
You may think the whole notion of it is silly, but the point is to intentionally limit the scope of your interaction concept so that you can focus on the details. It allows you to have a clear marker to make decisions with when developing the object and interaction. Any of the prepositions can work well, just pick one. As a reminder, you will only be able to list your preposition at the Exhibition Friday, so the interaction should be intuitive.

## Don't leave the design and fabrication of the physical component to the end
It may be tempting to work out all the code before considering the enclosure and object, but starting at the othe end has benefits. Understanding the specifics of the physical relationship between the person, sensor, and screen will provide you clear design directions. Even if it is rough, moving to the object will help you to design specific interactions with the sensor input and specific feedback on screen

## Start Simple and Iterate
One of the most effective strategies for developing any Physical Computing project is to start simple and iterate to polish & complexity. This is true for both the coding and the fabrication of your object. 
- Start with a simple/rough version of your physical design and the simplest version of the code you can make. 
- Get the interaction to work
- Make both of them better based on what you learned from the previous version
- Repeat steps as many times as you need
- - Be sure to save different versions of your software in case you have to go back to a previous one
- - Note that you are expected to write your own code for this project following the basic format & naming concepts we have been discussing in class.
