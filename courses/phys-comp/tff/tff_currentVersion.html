Introduction

Create a standalone device that uses the integrated LED matrix and either a standard HC-SR04 distance sensor or DIY pressure sensor to create interactive animations. This will result in a class Film Festival where all devices are displayed. The goal is to create an interactive animation on the LED Matrix that responds to input from the sensor. We will look at a variety of tools that allow you to create animations for the LED Matrix and how to link it to sensor input. We will also examine how raw sensor data can be customized for particular input using position, orientation, and data filtering methods. 

The goal is to create an experience that has a clear connection between the object, animations, and interaction method. 

Animation Tools and Controls

LED Matrix Editor

The LED Matrix Editor  is an online tool used to create animations and visual effects that can be shown on the LED Matrix. It allows you to create and manipulate frame by frame animations and convert them into code that can be shown on the arduino

Playback Control

The TinyFilmFestival library will be used to dynamically control, manipulate, and switch animations created in the Matrix Editor. This allow you to use sensor input to 

Set the playback type : Once, Loop, Boomerang

Set the play state : Play, Pause, Stop

Set playback properties : PlaySpeed

Set the animation to be displayed

Coding Concepts

This project will increase the complexity of the software created to design the relationship between the input data and results on screen.

Conditional Statements

if/else statements and  switch/case statements are a way to make decisions about incoming data. For example you can use these methods to play/pause an animation, change the playback type, or switch the animation being shown

Remapping Data

The map function as a way to relate 2 different data ranges. For example, you can use the distance a person is from the screen to control the speed of the animation.

State Machines

Situations in hardware and software are often described as modes or states. The concept allows for a different response to the same input. For example if the state of a machine is Off and you push the power button, it turns On. If the state is On and you push the power button, it turns Off. State Machines are a way in the code to manage this information both from the inputs, but also the current state of the media, ie : playing, paused, finished.

Interaction Design Concepts

Interaction as Preposition

Though code is the primary way to control the behaviour of your object, it is important that you can conceptually describe it as a preposition. Fundamentally, you are creating a relationship between the visual information on the LED Matrix and a person or thing. You should be able to describe that relationship through a simple preposition. For Example:

Spatial relationships: above, below, between, among, beside, behind, beneath, within, through

Movement/Direction: toward, across, along, around, into, onto, past, over, under

Time: before, after, during, since, until

Other relationships: with, without, except, like, despite, about, against

Always Be 'Alive'

When creating anything interactive, it is important to design it in a way that shows a person that it is 'Alive' and able to be interacted with. This is often seen in the design of game characters where they are never static to show that they are a dynamic part of the experience.

Material and Spatial Affordances

Another important aspect of creating interactive experiences is to create them in a way that it is clear how to engage with them. How might a person understand how to interact with it without detailed instructions? How does the material, scale, and location affect this?

 

Design Constraints / Requirements

  • Arduino, Sensor and all circuitry must be contained within an enclosure.
    • Enclosure can be a maximum of 30cm wide X 30cm deep. Height can be whatever is appropriate 
    • The device should be designed to be either free-standing, sitting on a table, or worn
    • Only the LED Matrix of the Arduino should be visible.
      • Consider how diffusion materials over the LED Matrix can change their visual effects
  • Each device must have a cable and battery/laptop to power it.
  • Write your own code. You can use the examples as a starting point, but you can't just use it verbatim 
  • The animation on screen must be "alive" at all times
    • When no-one is interacting with it, it cannot be static.
    • Consider at least 2 modes / states of the animation
