Late Work Policy

Project Light & Lightness
Overview

A critical part of any interactive project is to document it well. These works are always temporary, so it is important that you clearly capture the project for your portfolio. Documentation allows you to show the project in its best light, and also allows you to reflect on the process of making it. This isn't about proving that you made something, it is an opportunity for you to capture and archive the work.

Format

All documentation is to be created as a post on the course WordPress site.

WordPress Site - Log in using your OCADU credentials.

For guidance on how to create a post on WordPress, see the following tutorial: How to Make a Post on WordPress

Be sure to submit the URL of your post in Canvas.

Required Components

Your documentation post must include the following components:

Project Title

Title of your project.

Your Name

As you want it to appear on your portfolio.

Project Description (100 - 200 words)

  • Overall concept of the project and how the piece responds to light
  • NOTE: Don't give me unedited ChatGPT text. It always sounds ridiculous.

Link to the Code on Github

  • Do NOT copy/paste the code directly in your documentation post
  • How to Share Your Code

Main Project Image

A high quality image that best describes the work.

  • This must be set as the "Featured Image" in the post
  • It should also be included at the top of the post
  • The image should be well composed to show the project and interaction
  • Take care with the background and lighting

Main Project Video (10 - 20 seconds)

A high quality video that shows the piece responding to changes in light. Video should be uploaded to an external service (Vimeo, YouTube, Yuja, etc.) and embedded in the post.

  • This video should be taken outside of class to allow you to control the environment/sound/etc.
  • Use a tripod
  • Take care to control the background
  • You will likely have to take multiple videos and combine them
  • This can be split screen or linear editing
  • Pointing your phone camera at the Arduino while you are interacting with it is not sufficient
Post Settings
  • Post must have the Tag Light & Lightness
  • Post must have a Featured Image
  • Ensure that the Post has actually been Published
  • If the Post is created correctly it will show up on the Light & Lightness page on the website with an image
