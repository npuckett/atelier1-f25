Overview

AI coding tools have rapidly evolved and now represent a distinct skill set for developing code. Working with these tools is not simply "it does it for you"—there are specific methods and concepts that are now core to effective use.

Central to working with AI coding agents is the ability to clearly describe what you want to build. This is true whether you are working with an agent or not—clear project documentation is the foundation of successful development.

Even if your thesis project is not code-heavy, understanding these tools provides significant opportunity for creating things that were not previously possible within your timeline.

Preparation

Before the workshop:

  • Set up VS Code and GitHub Copilot using this guide: VS Code Setup Guide
  • Review the AI Coding Resources page for context on project documentation and agent instruction files
  • Think about one aspect of your thesis project that involves code or could benefit from a custom tool

Note: You get Copilot Pro free with your OCAD email. Other tools (Claude Code, Cursor) work similarly but may require separate accounts.

Activity

We will be working through the process of creating clear project documentation that can be used with AI coding agents—or with any development workflow.

Format: Individual or groups of 2-3

Choose ONE of the following to document:

Option A: Code for Your Project

An interactive component, visualization, or technical system that is part of your thesis work. This should attempt something beyond what you have done before.

Option B: A Tool That Helps You Make Your Project

A utility, generator, converter, or helper that would speed up your thesis development process. Think: asset generators, data processors, testing tools, or workflow automation.

Workshop Steps

Step 1: Brain Dump (10 min)

Write down everything you know about what you want to build in plain language. Don't worry about structure—capture features, constraints, uncertainties, and preferences.

Step 2: Structure with Template or Agent (15 min)

Use the Project Brief template (see Resources) OR ask an AI agent to help structure your brain dump. Review what it produces—you know your project better than it does.

Step 3: Peer Review (10 min)

Exchange documents with a partner. Can they understand what you're building? What questions do they have? What's unclear or missing?

Step 4: Refine Your PRD (15 min)

Based on feedback, fill gaps and clarify ambiguities. Add explicit constraints, examples, and edge cases. This is your Product Requirements Document.

Step 5: Start Building (remaining time)

Use your PRD to guide development with Copilot or another coding agent. Notice where your documentation helps and where you need to clarify further.

Deliverable

Submit a post containing:

  1. Your PRD — The structured project documentation you created (can be linked as a file or pasted as text)
  2. Link to your project/tool — A working prototype, GitHub repo, or hosted version
  3. Brief reflection — 2-3 sentences on how the documentation process affected your development workflow
Students taking 0.5 credit Thesis:
Complete the PRD and initial prototype. The code does not need to be fully functional, but should demonstrate progress toward your documented goals.
Students taking 1.0 credit Thesis:
Complete the PRD and a more developed prototype. Additionally, create a .github/copilot-instructions.md file (or equivalent) that captures your project's coding conventions. Include this in your submission.
Project Ideas by Medium

Not sure what to build? Here are starting points based on your thesis type:

Games / Interactive Experiences

  • A core mechanic prototype (one interaction loop, polished)
  • Level generator or asset randomizer
  • Save/load system or state manager
  • Dialogue or quest system tool

Installations / Physical Computing

  • Sensor data visualizer or debugger
  • Calibration tool for projections or sensors
  • State machine for installation modes
  • Data logger for user interaction patterns

Visual / Motion Media

  • Batch image processor or resizer
  • Color palette generator from reference images
  • SVG or graphics generator
  • Animation timing calculator

UX / App Projects

  • Interactive prototype (beyond Figma)
  • Data visualization dashboard
  • Form or survey with custom logic
  • API integration proof of concept
