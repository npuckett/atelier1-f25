# Game Design Component Resources

Sites similar to [component.gallery](https://component.gallery/components/) but for game design and 3D game components.

## UI & Visual References

### Game UI Database
**[gameuidatabase.com](https://gameuidatabase.com/)**

The closest equivalent in spirit to component.gallery. A massive reference tool for game UI and interface designers, featuring over 1,300 games and 55,000 UI screenshots. Browse and filter by category, animation, colour, material, layout, game genre, and more. Covers a deeply organized taxonomy of game components including HUDs, inventories, minimaps, settings screens, monetization flows, character editors, and dozens more. Focused on the 2D overlay/UI layer of games rather than 3D world-space components, but incredibly thorough.

## Programming Patterns & Architecture

### Game Programming Patterns
**[gameprogrammingpatterns.com](https://gameprogrammingpatterns.com/contents.html)**

The definitive reference for understanding the structural building blocks of a game engine. Catalogs patterns including Command, Flyweight, Observer, Prototype, Singleton, State, Double Buffer, Game Loop, Update Method, Bytecode, Subclass Sandbox, Type Object, Component, Event Queue, Service Locator, Data Locality, Dirty Flag, and Object Pool. Less visual than component.gallery but essential for the code/architecture side.

### Patterns Game Prog
**[patternsgameprog.com](https://www.patternsgameprog.com)**

A free book approach to learning design patterns through game programming, with Python and Pygame examples. Covers 2D strategy game development, OpenGL 2D facades, and more.

### Habrador — Game Programming Patterns in Unity
**[habrador.com/tutorials/programming-patterns](https://www.habrador.com/tutorials/programming-patterns/)**

A practical walkthrough of game programming patterns implemented in Unity with C# code. Covers Component, Event Queue, State, Double Buffer, Service Locator, Data Locality, Type Object, and more — with working game examples like a Minecraft clone.

## Game Design & Mechanics

### Game Design Library
**[nightblade9.github.io/game-design-library](https://nightblade9.github.io/game-design-library/)**

A curated catalogue of game-design links covering mechanics like crafting systems, boss design, feedback loops, health systems, side quests, metroidvania map design, and progression. Focused on the design-side "components" of a game rather than visual/UI.

### Game Mechanics: Advanced Game Design (Book)
**[peachpit.com/store/game-mechanics-advanced-game-design-9780321820273](https://www.peachpit.com/store/game-mechanics-advanced-game-design-9780321820273)**

By Joris Dormans & Ernest Adams. Includes a formal **Design Pattern Library** in its appendix covering mechanics patterns like static engines, dynamic engines, converter engines, engine building, static friction, dynamic friction, and escalation mechanics. Also covers the Machinations tool for visualizing and simulating game economies.

## Courses

### Holistic 3D — Design Patterns for Game Programming
**[learn.holistic3d.com/course/designpatterns](https://learn.holistic3d.com/course/designpatterns/)**

A hands-on Unity course by Penny de Byl covering the top 12 game programming design patterns with C# — commands, flyweight, observer, prototype, singleton, states, object pools, game loops, updates, and components. Includes 2D and 3D scenarios.

## The Gap

Nobody has yet built something as clean and browsable as component.gallery specifically for **3D game design elements** — camera systems, lighting rigs, physics interactions, navigation meshes, spawn systems, locomotion models, damage systems, etc. This feels like a gap waiting to be filled.
